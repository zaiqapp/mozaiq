# Mozaiq v2.2 â€” Data Sources Design

## Goal

Connect any widget to real data via CSV upload (snapshot) or Google Sheets (live polling). Users can map columns to widget fields manually or use AI to auto-map. The `DataSource` type is designed as a first-class object so future storage backends (Vercel Blob, separate DB table) can be swapped in without touching widget or UI code.

## Architecture

```
CSV upload  â†’ papaparse (client-side) â†’ rows stored in widget.dataSource.data (Postgres JSON)
Sheet URL   â†’ /api/data/google-sheets â†’ fetch public CSV export â†’ parse â†’ return rows
                                                    â†“
                              useDataSource(widget) hook
                              â”śâ”€â”€ CSV: returns widget.dataSource.data directly
                              â””â”€â”€ Google Sheets: fetches via route, polls every N seconds
                                                    â†“
                              Widget renderer reads resolved rows (falls back to mock config data)
```

Both source types share the same `mapping` object that maps widget field names to column names. Widget renderers receive resolved rows via a `dataSource` prop â€” they don't know or care whether the data came from CSV or Sheets.

## Tech Stack

- **papaparse** â€” CSV parsing (client-safe, no Node.js dependency)
- **Vercel AI SDK + AI Gateway** â€” map-columns and generate-widget endpoints
- **Next.js Route Handler** â€” Google Sheets proxy (avoids CORS on public export URLs)
- Existing: Prisma/Neon (dashboard JSON), `@clerk/nextjs` (auth), Zustand (store)

```ts
export type DataSourceType = 'csv' | 'google-sheets'

export interface DataSource {
  type: DataSourceType
  // CSV snapshot
  data?: Record<string, unknown>[]
  fileName?: string
  // Google Sheets live
  url?: string
  sheetName?: string
  refreshInterval?: number    // seconds; default 60, minimum 10
  lastFetched?: string        // ISO timestamp, set by useDataSource after each fetch
  // Shared
  mapping?: Record<string, string>  // widgetField â†’ columnName
  error?: string                    // last fetch/parse error message
}
```

`Widget.dataSource` stays optional â€” widgets without a connected source continue to use mock config data.

## New Hook â€” `hooks/useDataSource.ts`

Resolves a widget's data source into rows. CSV sources return immediately (data is inline). Google Sheets sources fetch via the proxy route and poll on `refreshInterval`.

```ts
export function useDataSource(widget: Widget): {
  rows: Record<string, unknown>[] | null
  isLoading: boolean
  error: string | null
  lastFetched: string | null
}
```

- Returns `null` rows if no `dataSource` is set (widget uses its own mock config data)
- On Google Sheets: fetches `/api/data/google-sheets?url=<encoded>` on mount and every `refreshInterval` seconds
- Caches last successful rows; on fetch error, keeps stale rows and sets `error`
- Updates `widget.dataSource.lastFetched` via `updateDataSource` store action after each successful fetch

## New File â€” `lib/parse-csv.ts`

Thin wrapper around papaparse for consistent config across upload and proxy parsing.

```ts
export function parseCsvText(text: string): {
  columns: string[]
  rows: Record<string, unknown>[]
  error?: string
}
```

- Uses `header: true`, `dynamicTyping: true`, `skipEmptyLines: true`
- Returns columns (header row keys) and typed rows
- Max 10,000 rows enforced â€” truncates with a warning in the return value

## New Route â€” `app/api/data/google-sheets/route.ts`

Server proxy to avoid CORS on Google Sheets public CSV export URLs.

```ts
// GET /api/data/google-sheets?url=<encoded-sheet-url>
export async function GET(req: Request)
```

- Accepts a full Google Sheets URL or spreadsheet ID
- Extracts the spreadsheet ID and constructs the export URL:
  `https://docs.google.com/spreadsheets/d/{id}/export?format=csv`
- Optional `gid` param for specific sheet tab
- Fetches, passes through `parseCsvText`, returns `{ columns, rows }`
- Returns 400 if URL is not a valid Google Sheets URL
- Returns 502 if the upstream fetch fails (sheet not public or not found)
- No auth required â€” public sheets only

## New Route â€” `app/api/ai/map-columns/route.ts`

Given a widget type, available columns, and sample rows, returns a suggested mapping.

```ts
// POST /api/ai/map-columns
// Body: { widgetType: WidgetType, columns: string[], sampleRows: Record<string, unknown>[], widgetFields: WidgetFieldDef[] }
// Response: { mapping: Record<string, string> }
```

- Uses `generateText` via AI Gateway (`anthropic/claude-haiku-4-5-20251001` â€” fast + cheap for this task)
- System prompt instructs the model to return only a JSON mapping object
- Validates response is a plain object with string values
- Requires auth (userId) â€” counts against rate limit? No â€” this is a utility call, not a dashboard generation. Not logged in `GenerationLog`.
- Returns 400 if required fields are missing

## New Component â€” `components/builder/DataSourcePanel.tsx`

Replaces the "Live data connections available in v2" placeholder in RightPanel. Rendered inside the Data section when a widget is selected.

### States

**No source connected:**
- Two buttons: "Upload CSV" and "Connect Google Sheet"
- One-line tooltip on each explaining snapshot vs live

**CSV flow:**
1. File input (accept `.csv`) â†’ parse client-side â†’ show preview (column names + first 3 rows)
2. "Use this data" button â†’ saves to `widget.dataSource`
3. Error state if parse fails or >10,000 rows

**Google Sheets flow:**
1. URL input + "Connect" button
2. Fetches via `/api/data/google-sheets` â†’ shows preview (column names + first 3 rows)
3. Refresh interval picker (10s / 30s / 1min / 5min / 15min / manual)
4. "Connect" button â†’ saves to `widget.dataSource`
5. Error state if sheet is not public or URL is invalid
6. "Last synced: Xs ago" indicator when connected

**Source connected (either type):**
- Source badge (filename or sheet URL domain)
- **Column mapping section** â€” one row per widget field from `widgetFieldRegistry`:
  - Field label + dropdown of available columns + "(none)" option
  - Selected mapping shown; unset required fields highlighted
- **"Auto-map with AI"** button â€” calls `/api/ai/map-columns`, fills dropdowns, user can adjust
- **"Disconnect"** link â€” clears `widget.dataSource`, reverts to mock data

## Widget Renderer Changes

`WidgetProps` gains an optional `resolvedData` prop:

```ts
export interface WidgetProps {
  config: WidgetConfig
  isSelected: boolean
  onSelect: () => void
  onDelete: () => void
  isReadOnly?: boolean
  resolvedData?: Record<string, unknown>[] | null  // null = use mock
}
```

`WidgetWrapper` (which renders all widgets) calls `useDataSource(widget)` and passes `resolvedData` down. Each widget renderer applies the column mapping to transform raw rows into its expected data shape.

**Chart widgets** (line, area, bar, donut, funnel): if `resolvedData` is present and mapping has `data` key, map the rows through the column mapping to produce `{ name, value }[]`. Fall back to `c.data` then mock.

**KPI**: if `resolvedData` is present, find the first row and extract mapped `value` and `change` columns. Fall back to `c.value` / `c.change`.

**Data table**: if `resolvedData` is present, use rows directly (columns auto-derived from mapping). Fall back to `c.rows`.

**Progress tracker / activity feed**: map rows through mapping to produce their expected item shapes.

**Text note**: if `resolvedData` is present and mapping has `content` key, use the value from the first row's mapped column as the text content.

**Gauge**: map `value` column from first row.

## Error Handling

| Scenario | Behaviour |
|----------|-----------|
| CSV too large (>10K rows) | Parse truncates at 10K, shows warning banner in DataSourcePanel |
| CSV parse error | Error message shown in DataSourcePanel, source not saved |
| Google Sheet not public | 502 from proxy â†’ "Sheet not found or not public" error in panel |
| Google Sheet fetch timeout | 5s server timeout â†’ 502 â†’ stale data kept, error banner on widget |
| AI map-columns failure | Toast "Auto-map failed â€” map manually", mapping unchanged |
| Column mapped but missing in data | Widget renders with empty/zero value, no crash |
